What the Server Does

The server is a lightweight FastAPI app that receives synced data from this client and serves a public live-results page at your domain. The client running at the event venue pushes race data every 30 seconds (or on demand).

The server only needs to run during race day. You can host it on any Linux VPS, a Raspberry Pi, or a cheap cloud instance (e.g. DigitalOcean $4/month droplet).

Option 1 — Docker (recommended)

Requires Docker installed on your server. SSH in and run:

# Pull the project onto the server
git clone https://github.com/your-org/mhm-race.git
cd mhm-race

# Set your API key (choose any random string)
echo "MHM_API_KEY=your-secret-key" > .env

# Start
docker compose up -d

The server will be available on port 8000. Point a reverse proxy (nginx/Caddy) at it or open port 8000 in your firewall for direct access.

Option 2 — Manual Python

# On the server (Python 3.11+ required)
git clone https://github.com/your-org/mhm-race.git
cd mhm-race
pip install -r requirements.txt

# Set env vars (or put in .env and source it)
export MHM_API_KEY=your-secret-key
export MHM_HOST=0.0.0.0
export MHM_PORT=8000

python run_server.py

To keep it running after you disconnect: use screen, tmux, or create a systemd service.

Systemd service (click to expand) 
[Unit]
Description=MHM Race Server
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/home/ubuntu/mhm-race
Environment="MHM_API_KEY=your-secret-key"
Environment="MHM_HOST=0.0.0.0"
Environment="MHM_PORT=8000"
ExecStart=/usr/bin/python3 run_server.py
Restart=always

[Install]
WantedBy=multi-user.target

Save to /etc/systemd/system/mhm.service, then: sudo systemctl enable --now mhm

Nginx Reverse Proxy (HTTPS)

server {
    listen 80;
    server_name results.mhmassacre.ca;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

Use certbot --nginx to add free HTTPS automatically.

Connecting the Client

Once the server is running, go to Settings and enter:

Server URL https://results.mhmassacre.ca (or your IP/domain)
API Key The same value you set as MHM_API_KEY on the server
Sync Interval 30 seconds is fine for race day

Dockerfile (for reference)

FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
ENV MHM_HOST=0.0.0.0
ENV MHM_PORT=8000
CMD ["python", "run_server.py"]

The docker-compose.yml exposes port 8000 and mounts a volume for the SQLite database so data survives container restarts. 

services:
  server:
    build: .
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data
    environment:
      - MHM_API_KEY=${MHM_API_KEY}
    restart: unless-stopped
